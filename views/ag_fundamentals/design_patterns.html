Moduler Design Pattern

JavaScript modules are the most prevalently used design patterns for keeping particular pieces of code independent of other components.
Angular Uses this Design pattern as follows.
angular.module('module-name',[])


Model View Controller

The name of the pattern consists of the names of its actors: Model - stores an application data; View - renders Model for a client; and Controller - updates Model by reacting on client's actions.
  • Model:Represents the Domain Data
  • View: Represents the Html page
  • Controller : Processes and responds to events, typically user actions, and invokes changes on the model and perhaps the view(In Simple it represents Javascript)

Single Ton

The Singleton Pattern limits the number of instances of a particular object to just one. This single instance is called the singleton.


  1. var Singleton = (function () {
  2. var instance;
  3.  
  4. function createInstance() {
  5. var object = new Object("I am the instance");
  6. return object;
  7. }
  8.  
  9. return {
  10. getInstance: function () {
  11. if (!instance) {
  12. instance = createInstance();
  13. }
  14. return instance;
  15. }
  16. };
  17. })();
  18.  
  19. function run() {
  20.  
  21. var instance1 = Singleton.getInstance();
  22. var instance2 = Singleton.getInstance();
  23.  
  24. alert("Same instance? " + (instance1 === instance2));
  25. }


Two Way Data Binding

Angular Js Two way data binding synchronizes data(Model) between View(HTML) and Controller(Js) .
View(Html) is responsible of representation of model(data) , any modification in view reflected to model and any modification to model reflected back to view(html).


Dependency Injection

dependency injection is a technique whereby one object supplies the dependencies of another object
An injection is the passing of a dependency to a dependent object (a client) that would use it.
